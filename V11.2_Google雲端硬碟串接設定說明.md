# V11.2 Google Drive 固定綁定 — Google Cloud Console 設定步驟

這份文件是給**管理者**看的操作手冊，不是程式碼文件。目的是取得系統需要
的 4 個環境變數：`GOOGLE_OAUTH_CLIENT_ID`、`GOOGLE_OAUTH_CLIENT_SECRET`、
`GOOGLE_OAUTH_REDIRECT_URI`、`GOOGLE_TOKEN_ENCRYPTION_KEY`。

⚠️ 這一步必須由您本人操作（登入您的 Google 帳號、同意 Google 的服務條款、
承擔這個 Google Cloud 專案未來的 API 配額與帳單責任）——我這邊沒有辦法
代替您建立，也不會自己生成 Client ID/Secret。

---

## 前置：您需要先知道系統的完整網址

Google 要求「重新導向 URI」必須是一個真實存在、逐字精確的網址。**在您
還沒有把這個系統部署到一個實際網址（例如 Render 給的
`https://sanxuan-erp-xxxx.onrender.com`，或您自己的網域）之前，沒辦法
完成下面「步驟四」**。如果您還沒有實際網址，建議：

1. 先照 `render.yaml` 的說明把這個專案部署到 Render（或您選擇的其他平台），
   拿到一個實際網址。
2. 部署時先把環境變數 `SKIP_PRE_DEPLOY_BACKUP` 設成 `"1"`（`render.yaml`
   裡已經預先這樣設定），這樣即使還沒有連結 Google Drive，系統也能先
   正常啟動。
3. 拿到實際網址之後，再回來完成下面的步驟。

假設您的網址是 `https://sanxuan-erp-xxxx.onrender.com`，那麼這個系統的
OAuth callback 完整網址就是：

```
https://sanxuan-erp-xxxx.onrender.com/api/system-center/google-drive/oauth-callback
```

（`/api/system-center/google-drive/oauth-callback` 這一段路徑是程式碼裡
固定寫死的，請直接照抄，不要修改。）

## 步驟一：建立 Google Cloud 專案

1. 前往 https://console.cloud.google.com/
2. 用您要綁定備份的那個帳號登入（例如 fa0225234163@gmail.com）。
3. 左上角「選取專案」→「新增專案」，專案名稱可以取
   「三玄宮ERP備份」之類好辨識的名字，按「建立」。

## 步驟二：啟用 Google Drive API

1. 左側選單「已啟用的API和服務」→「+ 啟用API和服務」。
2. 搜尋「Google Drive API」，點進去按「啟用」。

## 步驟三：設定 OAuth 同意畫面

1. 左側選單「OAuth 同意畫面」（或新版介面的「Google Auth Platform」→
   「品牌宣傳」）。
2. 使用者類型選「外部」（除非您的機構有 Google Workspace 網域，選「內部」
   會更單純，但一般宮廟不一定有）。
3. 應用程式名稱填「三玄宮ERP」，使用者支援電子郵件與開發人員聯絡資訊
   填您自己的 email。
4. 範圍（Scopes）這一步先不用加，程式碼會在授權連結裡自動帶
   `https://www.googleapis.com/auth/drive.file`（只存取這個系統自己
   建立的檔案，不是完整雲端硬碟權限）。
5. 測試使用者：如果應用程式狀態還是「測試中」（尚未送 Google 審核），
   這裡務必加入 fa0225234163@gmail.com，否則這個帳號授權時會被拒絕。
   （備份功能只有您內部使用、不對外公開，通常不需要特地送 Google 審核
   成「正式版」——除非 Google 因為使用的 scope 判定需要驗證，若跳出
   相關提示，屆時再依畫面指示處理即可。）

## 步驟四：建立 OAuth 用戶端（Client ID / Secret）

1. 左側選單「憑證」→「+ 建立憑證」→「OAuth 用戶端 ID」。
2. 應用程式類型選「網頁應用程式」。
3. 名稱填「三玄宮ERP網站服務」。
4. 「已授權的重新導向 URI」按「新增 URI」，貼上前置步驟算出來的完整
   網址，例如：
   ```
   https://sanxuan-erp-xxxx.onrender.com/api/system-center/google-drive/oauth-callback
   ```
5. 按「建立」。畫面會跳出「用戶端 ID」與「用戶端密碼」——**請立刻複製
   保存**，密碼之後無法再次完整顯示（忘記的話要重新產生一組）。

## 步驟五：把憑證設定成環境變數

到您部署系統的環境變數設定頁面（例如 Render 網站服務的
「Environment」分頁），新增：

| 環境變數 | 值 |
|---|---|
| `GOOGLE_OAUTH_CLIENT_ID` | 步驟四拿到的用戶端 ID |
| `GOOGLE_OAUTH_CLIENT_SECRET` | 步驟四拿到的用戶端密碼 |
| `GOOGLE_OAUTH_REDIRECT_URI` | 跟步驟四填的重新導向 URI **逐字相同**（協定/網域/路徑，包括有沒有結尾斜線都要一樣） |
| `GOOGLE_TOKEN_ENCRYPTION_KEY` | 見下方「產生加密金鑰」 |
| `BACKUP_CRON_SECRET` | 見下方「產生排程密鑰」 |

### 產生加密金鑰（`GOOGLE_TOKEN_ENCRYPTION_KEY`）

這組金鑰用來加密存在資料庫裡的 Google refresh token（見
`src/lib/googleDrive.ts` 的說明），不能隨便打幾個字充數，需要真正隨機
的 32 bytes。如果您的電腦或 Render 的 Shell 分頁能執行指令，跑一次：

```
openssl rand -hex 32
```

會印出一串 64 個字元的十六進位亂數，例如
`3f9a1c...`（實際會是完全隨機的一串），把整串複製當作這個環境變數的值。

### 產生排程密鑰（`BACKUP_CRON_SECRET`）

同樣用 `openssl rand -hex 32` 產生一組（跟上面那組不要用同一個值），
這組是給外部排程服務呼叫「每日/每週/每月自動備份」API 時用來驗證身分
的密鑰，設定方式見 `render.yaml` 裡「每日/每週/每月自動備份的排程觸發
方式」那一段說明。

## 完成後怎麼用

環境變數都設定好、系統部署完成後：

1. 用最高管理員身分登入（在畫面上方選擇操作人員），進入
   【系統管理】→【Google Drive連線】。
2. 按【連結Google Drive】，會跳轉到 Google 的登入/同意畫面，登入
   fa0225234163@gmail.com 這個帳號並同意授權。
3. 授權完成後會自動導回系統，畫面顯示「已連結：fa0225234163@gmail.com」。
4. 之後系統會固定使用這個帳號備份，不受您電腦上 Chrome 目前登入哪個
   Google 帳號影響（因為授權資訤是存在系統的資料庫裡，不是存在瀏覽器）。

## 常見問題

**Q：授权时 Google 说「此应用未经过 Google 验证」，可以继续吗？**
A：如果應用程式狀態還是「測試中」，只要 fa0225234163@gmail.com 已經
加進步驟三的「測試使用者」名單，會有一個「進階」或「繼續前往（不安全）」
的連結可以點下去繼續，這是正常現象（因為這個 App 只給自己內部用，
沒有送 Google 審核），不是系統有問題。

**Q：換了網域/URL 之後，Google Drive 連線會失效嗎？**
A：會需要回到步驟四把新的網址加進「已授權的重新導向 URI」清單裡（可以
同時保留舊的），並把 `GOOGLE_OAUTH_REDIRECT_URI` 環境變數改成新的網址；
已經完成的授權（refresh token）不會因此失效，只有「之後要重新走一次
連結流程」時才需要網址正確。

**Q：這組憑證的 API 配額/費用會很貴嗎？**
A：Google Drive API 對這種等級的使用量（一天幾次上傳/下載）通常在免費
配額範圍內，不會產生費用；但配額與帳單設定屬於您 Google Cloud 帳號的
責任範圍，這裡不代您做出保證，如有疑慮建議在 Google Cloud Console 的
「帳單」頁面確認目前方案。
