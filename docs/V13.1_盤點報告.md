## 十一、我必須誠實說明的三件事

**1. 這份規格的實際體量。**
第十一節「年度燈」在目前專案是**從零開始的新模組**——`ActivityType` 有 `GUANGMING_LANTERN` / `TAISUI_LANTERN` / `FAMILY_LANTERN` 三個 enum 值，但沒有任何畫面、API、列印模板存在；`燈牌`／`疏文` 目前只是 `AdditionalPrintItemType` 的兩個標籤，沒有實作。這一節本身就相當於過去一整輪版本的工作量。

加上第十節（活動年度控制）、第十二節（全列印國字化）、第一節（身分證），這份規格實質包含**四個彼此獨立的大模組**。妳要求「一次完成、不得零碎修改」，我理解用意是避免反覆返工；但把這四個模組壓進同一個 commit，一旦 Render 部署出問題，無法單獨回退任何一塊，正式系統會整套卡住。

**2. 第十五節與其他節本質不同。**
它是對正式資料庫的破壞性操作，不是程式修改。依妳自己的規則「所有重大修改必須等待確認」，這一節不能包在自動流程裡。

**3. 沙盒無法執行 `npm run build`。**
`prisma generate` 對 linux-arm64 回 403，這是環境限制、每一輪都會發生。TypeScript 我可以驗（`tsc --noEmit`），Build 與 Prisma 產生必須在妳的 Mac 上跑。
