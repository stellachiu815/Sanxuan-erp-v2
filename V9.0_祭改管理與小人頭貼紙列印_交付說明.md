# V9.0「祭改管理與小人頭貼紙列印」交付說明

這一輪依照你給的完整規格（十六節），直接把資料表、後端邏輯、前端畫面、
權限、列印模板、PDF 產生、自動測試全部完成，不是規格文件。

---

## 一、這一輪做了什麼

### 1. 祭改年度／報名資料

- 台北三玄宮只有一種祭改，每年建立一筆《祭改年度》（`PurificationYear`），
  例如「民國一一五年度祭改」——年度名稱用「逐字讀法」（跟門牌號碼同一套
  規則），不是「民國一百一十五年」。
- 每一筆報名（`PurificationRegistration`）都關聯到信眾主資料
  （`Member`）跟家戶（`Household`）；**姓名／性別／生日一律即時從信眾
  主資料讀出，地址／電話從家戶讀出，不會另外存一份會脫鉤的複本**——只有
  「臨時報名者」（信眾主資料還沒建立的人）才會用到手動輸入欄位。
- 收款狀態／金額／備註／報名日期／狀態（有效／取消／補報）／是否已列印／
  列印日期／列印批次都有對應欄位。

### 2. 性別與建生／瑞生

- 信眾主資料的 `gender` 欄位（男／女／未填寫），列印時自動換算：男性→
  「吉時建生」、女性→「吉時瑞生」、未填寫→列入「待確認清單」，系統不會
  自己猜。

### 3. 歲數（虛歲）

- 歲數**不存欄位**，每次查詢/列印當下依「出生農曆年＋祭改年度」重新計算，
  絕對不會變成固定寫死的數字，跨年度會自動加一歲。出生年份不完整/無法
  計算時，同樣列入待確認清單。

### 4. 中文國字轉換（兩套系統，不能混用）

- **數值讀法**（歲數/生日月日）：54 → 五十四、7 月 7 日 → 七月七日
  （不會變成「初七」）。
- **逐字讀法**（地址門牌／年度名稱）：181 → 一八一（不是一百八十一）。
- 農曆日期一律用完整國字，不會自動改成「初七」「廿五」這類口語簡寫
  ——這跟系統既有、給其他畫面用的 `formatLunarDate()`（會用初一/廿五）是
  刻意分開的兩套邏輯，不會互相影響。

### 5. 祭改編號自動編列

- 報名完成立即自動編號，接續目前最後一個有效編號；自動跳過任何「包含
  連續 44」的號碼（44/144/244/344/440-449/1440……），編號 125 這種橫式
  數字不會被拆開。
- 額外禁用號碼由管理者在「禁用編號設定」頁維護，「連續 44」規則本身是
  寫死的程式邏輯，不會出現在資料表裡、也不能被移除。
- 編號一旦開始列印即鎖定：取消保留原編號、不重排、不重用；鎖定後新增
  的報名者自動變成「補報」，接續編號。只有尚未列印時，管理者才能執行
  「重新編號」，畫面上需要兩次確認才會真正執行。

### 6. A4 小人頭貼紙（3欄×11列＝33格）

- 固定 A4、固定 3 欄 × 11 列，每格尺寸/格距/裁切位置寫死在
  `src/components/purification/stickerSheet.ts`，程式不會因為內容多寡
  拉伸格子或改變紙張尺寸。
- 每格直式三欄、由右至左：右欄編號（橫式嵌字）＋姓名、中欄歲數／農曆
  生日／建生瑞生、左欄地址（完整不截斷）。
- 智慧字級最佳化（`src/lib/purificationLayout.ts`）：姓名/中間欄/地址欄
  各自獨立縮放，姓名依「二字放大／三字標準／四字微調／五字以上才縮小」
  分級，縮到最小仍放不下才會列入「人工確認清單」，絕不硬擠、截字、
  重疊，也不會把直式改成橫式。

### 7. 列印預覽與列印批次

- 【產生預覽】：純查詢，不會標記已列印、不會鎖定年度，可以反覆調整
  篩選條件重新預覽。
- 【最佳化版面】：顯示「已自動調整幾筆／哪幾筆縮小字體／哪幾筆仍需
  人工確認」摘要。
- 【產生列印批次】：這一步才會真正標記已列印、視需要鎖定年度，只要有
  任何一筆沒通過列印前檢查（性別/生日/地址/編號/重複等），整批擋下，
  不會漏印。
- 支援全部列印／尚未列印／指定編號範圍／指定姓名／單一格放大預覽；
  補印單筆／補印整張／重新產生 PDF 也都是同一套流程。

### 8. 資料沿用（沿用去年）

- 只複製參加者／地址／備註／家戶關係；歲數/編號/收款/列印紀錄一律重設，
  在新年度重新計算/重新編列。
- 提供差異比對：新增／去年已取消（不會複製過來）／地址或生日可能異動
  （信眾主資料在去年報名之後被修改過，提示人工核對）／性別待確認。

### 9. 自動測試

63 個測試全部透過 Node 內建 `node:test`（`npx tsx --test tests/*.test.ts`）
真的跑過一次，不是憑空宣稱：

```
# tests 63
# pass 63
# fail 0
```

涵蓋：男性/女性/性別未填、二～五字以上姓名分級、最長地址/地址未填、
農曆七月七日正確顯示（不會變初七）、虛歲跨年度自動加一、編號 43→45
跳過 44、跳過 144/244/344/440-449、編號 125 保持橫排、33 筆剛好一張/
34 筆自動產生第二張、取消不重排、補報接續最後編號、無截斷/無重疊/
無超出貼紙範圍等等。

## 二、誠實說明：這一輪的限制

### 1. 官方「113小人頭1-33」Word 範本檔案沒有實際送達

這次開發過程中確認，你提到的 Word 範本檔案並沒有真的附加到對話裡。依照
你的指示（「除非我另外要求，否則不要完全照抄版面，而是保留目前尺寸、
欄位位置與列印規格，並依我們之前確認的規則建立智慧列印模板」），這一輪
用**合理預設值**建立了完整可運作的智慧列印模板：

- 貼紙留白／格距（`stickerSheet.ts` 的 `STICKER_A4_PAGE`）。
- 三欄各字級的 pt 數與容納字數（`purificationLayout.ts` 的
  `COLUMN_FONT_TIERS`）。

之後範本檔案送到後，**只需要調整這兩個檔案裡的數字**，不需要重寫任何
功能——3欄×11列的固定排版、直式右到左的閱讀順序、編號橫式嵌字的做法，
都是需求明確要求「不論範本長怎樣都不能變」的部分，這次已經照做，之後
也不會受範本影響。

### 2. 沙盒環境沒有資料庫、無法真正建表/寫入資料

跟前面幾輪一樣，這個沙盒沒有 PostgreSQL、無法連線 GitHub/Render、無法
`npm install`（`@prisma/client`、`lunar-javascript` 都無法真的載入），所以：

- 完全靠 Prisma schema 手動比對資料表結構、手動核對每個程式裡用到的
  欄位名稱都真的存在（已完成，詳見下方驗證清單），但**沒有真的對著
  資料庫執行過**任何一支 CRUD。
- `purification.ts` 裡跟資料庫/`lunar-javascript` 有關的程式碼（信眾/
  家戶關聯查詢、農曆換算），只做到了「靜態檢查通過」，還沒有「實際執行
  通過」——這點只有真正部署到有資料庫的環境才能驗證。
- 5 個不依賴 Prisma/lunar-javascript 的純邏輯模組（中文數字轉換／編號
  規則／版面最佳化／虛歲計算／列印前檢查）**是真的執行過**，63 個測試
  全部通過，這是刻意的架構決定——把最容易出錯的規則寫成可以在沙盒裡
  真正驗證的形式。

### 3. PDF/實體列印無法在沙盒裡驗證

PDF 匯出沿用既有牌位列印（V4.1）已經在用的 `html2canvas` + `jsPDF`
做法，程式邏輯一致，但這個沙盒不能開瀏覽器、不能實際產生 PDF 檔案來看，
也沒有真正的貼紙紙材可以核對裁切位置——這需要部署後，拿實際印表機/
貼紙紙材測試校正（畫面上已經預留「印表機校正偏移」的概念空間，但實際
校正數值要等真正列印過才能調整）。

### 4. 「一般工作人員不可修改禁用編號」還沒辦法在後端真正擋下

跟 V8.0 的回收區還原/永久刪除一樣的限制：系統目前沒有登入/session
機制，`getCurrentUser()` 永遠回傳 `null`，`src/lib/permissions.ts` 已經
定義好 `assertPurificationPermission()` 規則（只有 `SUPER_ADMIN` 能操作
禁用編號清單），但因為沒有真正的登入身份可以驗證，這支規則暫時**沒有**
接到 API route 上做強制擋下（接了會讓整個功能在目前沒有登入系統的狀態
下完全無法使用）。目前只能：畫面上把「禁用編號設定」入口獨立成一個
單獨頁面，並在頁面上明確提醒「這個設定頁只應該給管理者使用」。等登入
機制做出來，只要在 `src/app/api/purification/banned-numbers/route.ts`
接上 `getCurrentUser()` + `assertPurificationPermission()` 就會生效，
規則本身不用重寫。

### 5. 「去年與今年差異比對」裡的地址/生日變更是「提示」，不是「精確舊值比對」

因為系統刻意不重複儲存姓名/性別/生日/地址（一律即時引用信眾主資料），
「地址變更」「生日變更」沒辦法像一般系統一樣比對「去年存的值 vs 今年
存的值」——因為兩邊本來就是同一份即時資料。這裡改用誠實的替代方案：
如果信眾/家戶資料是在「去年那筆報名建立之後」才被編輯過，就提示「可能
異動，請人工核對」，而不是假裝系統知道一個它其實沒有儲存過的歷史數值。
詳見 `src/lib/purification.ts` 裡 `buildPurificationYearDiffs()` 的說明
註解。
